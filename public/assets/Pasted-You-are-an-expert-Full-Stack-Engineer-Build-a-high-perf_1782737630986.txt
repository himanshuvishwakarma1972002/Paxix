You are an expert Full-Stack Engineer. Build a high-performance corporate website inspired by paixservices.com using Next.js (App Router), JavaScript, Node.js, and Tailwind CSS. The app must fetch dynamic page content from Sanity CMS and include a contact form that logs user inquiries into a PostgreSQL database via a Node.js serverless route.

### 1. Project Stack & Architecture
- Framework: Next.js (App Router, JavaScript)
- Styling: Tailwind CSS (Clean corporate theme: deep navy primary text, white/light-gray backgrounds, teal/cyan accents)
- CMS Integration: Sanity.io (for all marketing pages, hero text, services list, testimonials, and education course details)
- Database: PostgreSQL (for persistent user leads, contacts, and form submissions)
- Lucide React for consistent iconography.

### 2. Sanity Studio Integration & Schema Requirements
Create a Sanity schema setup (`/sanity/schemas/`) with the following document types to feed data into the frontend:
- `pageContent`: Document to handle core copy (title, subtitle, heroText, features array, stats array).
- `solution`: Document for services (title, description, category ['Service' or 'Technology'], iconSlug).
- `testimonial`: Document for client quotes (quote, author, authorRole).
- `course`: Document for the Education page (title, description, certifications array like CPC/CCS, features).

Ensure the Next.js frontend fetches this data using `@sanity/client` or `next-sanity` through optimized `groq` queries. Implement simple preview states or clean fallbacks if Sanity data is loading.

### 3. PostgreSQL Database Setup
Create an internal Next.js API route (`/api/contact`) or standalone database connection module using `pg` (node-postgres) to handle lead generation:
- Database table name: `contact_submissions`
- Columns: `id (SERIAL PRIMARY KEY)`, `name (VARCHAR)`, `email (VARCHAR)`, `phone (VARCHAR)`, `message (TEXT)`, `created_at (TIMESTAMP)`.
- The API route must securely parse incoming JSON from the Contact Form, write it to the PostgreSQL database, and return a 201 Success response.

### 4. Page UI & Structure Layout
- Navigation Bar: Sticky, white background with a modern blur effect, high-contrast links, and a primary teal 'Get Started' action button.
- Home Page (`/`): 
  * Hero Section: Bold value proposition headline ("Intelligence that Simplifies Care"), subtext, and split dual CTAs.
  * Stats Ribbon: A grid displaying impact metrics (e.g., "100+ Hospitals", "98% Success Rate").
  * Dual Cards Segment: Side-by-side components highlighting 'Service Solutions' (RCM, Billing Automation) and 'Technology Solutions' (EHR, Analytics).
- About Page (`/about-us`): Split-screen story layout, core company values grid, and a modern carousel/stack of customer testimonials.
- Education Page (`/education`): Clean list/grid cards of training modules (Medical Coding, RCM Training), certification tags (CPC, CCS, CPMA), and an emphasis on career paths.
- Contact Page (`/contact`): Left side contact information and structural details; Right side interactive validation form (Name, Email, Phone, Message) connected directly to your `/api/contact` endpoint.
- Footer: Clean 4-column multi-link directory layout with a privacy statement, terms of use, and dynamic copyright year.

### 5. Implementation Workflow Instructions
1. Scaffold the boilerplate structure with Tailwind configuration.
2. Build the PostgreSQL connection script and the contact form route.
3. Establish the Sanity config file (`client.js`) and mock out the GROQ fetch functions.
4. Program layout components (Navbar, Footer) ensuring fully responsive layout structures (mobile drawer menu included).
5. Build the pages using the fetched Sanity schemas, rendering beautiful UI blocks with clean typography, smooth transitions, and proper loading skeletons.
